Level 9 -> Level 10

May 6, 2020 1-minute read

Level Goal

The password for the next level is stored in the file data.txt in one of the few human-readable strings, preceded by several ‘=’ characters.

Solution

Let’s first check the data type of the file we are given:

bandit9@bandit:~$ file data.txt
data.txt: data

Short Lesson

The .data file extension is given to the data files that have been created by and used for a variety of different software applications. These files can be used on the Linux, Unix and MAC operating systems. The majority of DATA files are saved in the binary file format. DATA files can contain a variety of different file types including document files, project files, libraries and other different file types.

In order to read the content of such files, we can use the strings command, which is mainly useful for determining the contents of non-text files. We are also searching for lines containing several “=” characters, so we can pipe the strings command with a grep one:

bandit9@bandit:~$ strings data.txt | grep ====
========== the*2i"4
========== password
Z)========== is
.========== truKLdjsbJ5g7yyJ2X2R0o3a5HQJFuLk    <-- This is the password for level 10

 

We successfully found the password for Level 10 !
